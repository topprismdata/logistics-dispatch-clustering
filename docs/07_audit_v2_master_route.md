# 阶段 1.5 v2 审计报告

## ⚠ 结论先行: Master Route (ConVRP) 假设也失败

3 轮自审 + agy 独立交叉验证**一致认定**: 设计 v2 (Master Route / ConVRP) 在DC_B 数据上**不成立**。

| 假设 | 实测 | 判定 |
|---|---|---|
| 高频 anchor 形成稳定骨架 | 90.1% 含 anchor 路线只有 1 个 anchor | ❌ |
| anchor 共现矩阵密集 | 稀疏度 88.7% (102/903 实际共现) | ❌ |
| anchor 携带其他客户 | anchor PC = 100% 路线 PC (独占) | ❌ |
| 多客户路线有稳定共现 | median = 1 次, 强共现 ≥5 次只 2.9% | ❌ |

**所有 4 个 ConVRP 假设均不成立**。

---

## 1. 3 轮自审发现

### Round 1 — anchor 定义 + Hard Mode threshold sweep

```
Anchor 定义审查 (43 客户, >50 次配送)
  含 anchor 路线中只 1 个 anchor: 90.1% (1812/2010)
  anchor 共现矩阵稀疏度: 88.7% (903 对中只 102 实际)
  平均共现次数: 2.9 (统计噪声)

Hard Mode Threshold Sweep (验证 thr=0.5 是否最优)
  thr=0.3: ARI=0.002, P=6.1%, avg_cluster=19.8 (过大)
  thr=0.5: ARI=0.015, P=6.8%, avg_cluster=3.8 (最接近真实)
  thr=0.7: ARI=0.003, P=9.1%, avg_cluster=1.1 (过碎)
  thr=0.9: ARI=0.000, P=0.0%, avg_cluster=1.0
```

→ **任何 threshold 都无法救活 Siamese Hard Mode**。

### Round 2 — `greedy_cluster_from_pairs` 函数正确性 + "大客户独占"

```
greedy_cluster_from_pairs 测试: A-B-C 强 / D-E / F 单独
  实际输出: {'A':0, 'B':0, 'C':0, 'D':1, 'E':1, 'F':2}  ✓ 正确

单 anchor 路线中 anchor PC 占路线 PC 比例 (n=2564)
  median: 100.0%
  p25: 100.0%
  p75: 100.0%
  mean: 100.0%
```

→ **anchor 一旦出现就独占一车 (PC=100%)**。这不是"骨架",是 SOP-1 直配。

### Round 3 — 高 PC vs 高频是不同群体 + 多客户路线无结构

```
高频客户 (>=50 次): 43 个
高 PC 客户 (avg PC > 200): 133 个
两者重叠: 只 20 个
高 PC 但低频: 113 个 ("偶尔大单")
高频但低 PC: 23 个 ("频繁小客户")

多客户路线 (n=2333) 共现统计:
  共现对总数: 45,437
  共现次数分布: max=20, p99=6, p90=3, median=1
  强共现对 (>=5 次): 1,317 (2.9%)
```

→ **DC_B 没有稳定结构**。多客户路线是"动态随机拼车",不是"查表"。

## 2. agy 独立交叉验证

agy 给出**与 3 轮自审一致的严厉判断**:

### agy 5 个核心结论

> **1. ConVRP "anchor 共现骨架" 完全不成立**
> 90.1% 含 anchor 路线仅 1 个 anchor; 88.7% 矩阵稀疏度; 平均共现 2.9 次 (噪声)。
> "Master Route 依赖的'高频 anchor 稳定共现形成骨架'假设在DC_B **完全脱离实际**。"

> **2. anchor 独占一车 (100% PC) 本质是直配 (Point-to-Point), 退化为 SOP-1**
> "这些高频 anchor 实际上是稳定的大单直配客户。强行放入 ConVRP 骨架寻优,是把简单的直发逻辑 (SOP-1) 复杂化,**无任何算法增益**。"

> **3. 多客户路线 median=1 = 动态随机拼车**
> "订单是随机动态的。每次排线根据当天订单量、位置、运力**动态拼凑最优解**,不存在长期固化的拼车默契。ML 查表逻辑无法奏效。"

> **4. ML 方向应转向: 动作评估 / 时间预估 / 需求预测**
> 不学"路线拓扑",改学:
> - **RL 动作评估**: SOP-1 vs 动态拼车的临界分裂点决策
> - **GNN 时间预估**: 卸货等待时间预测 (尤其针对 113 个偶尔大单客户)
> - **大单需求预测**: 提前半天预测高 PC 低频客户下订概率

> **5. 阶段 1.5 应放弃 ML 路线学习, 走纯规则/启发式**
> Siamese (ARI=0.01) + ConVRP 相继失败 = 数据缺乏静态结构。
> 新设计: 规则分流 SOP-1 → ALNS/LNS 启发式拼车 → ML 仅作辅助参数输入层。

## 4. 重大方法学教训

### 教训 1: 不要把"频次"等同于"骨架"
- 高频 ≠ 稳定搭档
- 高 PC + 高频才接近"大客户独占" (SOP-1),但这不是 ConVRP anchor

### 教训 2: 不要假设数据有静态结构
- DC_A 共现率 100% → 查表思维
- DC_B 共现率 24.8% + median=1 → 动态拼车
- **两种数据本质不同, 不能套用同一方案**

### 教训 3: ML 的正确角色
- 不是"决策器" (Hard Mode ARI=0.01 证明)
- 而是"参数预测层":
  - SOP-1 临界点预测 (PC 阈值动态化)
  - 卸货时间预估 (GNN 时空预测)
  - 大单需求预测 (时序预测)

### 教训 4: Hard Mode 才是真实评估
- Easy Mode (83%) 是幻觉
- Hard Mode (ARI 0.01) 才是真相
- 任何 future baseline 必须先过 Hard Mode 关

## 5. 阶段 2 真正方向 (基于审计)

按 agy 建议 + 我的判断, 放弃 ConVRP/Siamese/Pair-prediction, 改为:

### 架构 v3: 规则分流 + 启发式 + ML 辅助

```
┌────────────────────────────────────────────────────┐
│  Stage 0: SOP-1 规则分流 (纯规则, 无 ML)           │
│  - 当日订单 PC > 容量阈值 (动态) → 直接 SOP-1 直配 │
│  - 剥离 ~40% 大单路线 (含 anchor + 偶尔大单)        │
└────────────────────┬───────────────────────────────┘
                     ▼
┌────────────────────────────────────────────────────┐
│  Stage 1: ALNS / LNS 启发式拼车 (经典 OR)          │
│  - 剩余小单客户 (~60%) 动态拼车                    │
│  - 容量约束硬约束 (mask)                            │
│  - 目标: 最小化总距离 + 平衡装载率                 │
└────────────────────┬───────────────────────────────┘
                     ▼
┌────────────────────────────────────────────────────┐
│  Stage 2: ML 辅助层 (不作决策, 只预测参数)         │
│  - GNN 时空模型: 卸货时间预估 (送给 Stage 1)       │
│  - 时序模型: 大单需求预测 (送给 Stage 0 预留运力)  │
└────────────────────────────────────────────────────┘
```

### 阶段 2 任务

| # | 任务 | 类型 |
|---|---|---|
| **N1** | SOP-1 规则分流 baseline (PC>260 阈值) | 纯规则 |
| **N2** | OR-Tools CVRP / ALNS 启发式 baseline | 经典 OR |
| **N3** | GNN 卸货时间预估 (用历史卸货时间数据) | ML 辅助 |
| **N4** | 时序大单需求预测 (用 PC 时间序列) | ML 辅助 |
| **N5** | Hard Mode 评估所有 baselines (ARI + Partition F1) | 评估 |

**预期结果**:
- N1 SOP-1 分流: ARI ~0.3-0.5 (剥离大单后剩余任务变简单)
- N2 ALNS: ARI ~0.4-0.6 (经典启发式优势)
- N3+N4 ML 辅助: 在 N1+N2 基础上 +5-10% ARI 提升
